COMO PROBAR (Angular + Docker Backend/BD)
=========================================

Objetivo
--------
Levantar rapidamente:
1) Backend Spring + PostgreSQL con Docker Compose
2) Frontend Angular en local

Este documento refleja el flujo ACTUAL del proyecto (sin frontend historico Flask).


Requisitos
----------
- Docker Desktop encendido
- Node.js y npm instalados


Lanzamiento rapido (copy/paste)
-------------------------------
Ejecuta exactamente estos comandos en este orden.

TERMINAL A (backend + bd con Docker):
cd .
docker compose up --build -d
docker compose ps

TERMINAL B (frontend Angular):
cd .\juegos1000tres-frontend
npm install
npm start

Abrir en navegador:
- Frontend: http://localhost:4200
- Backend (host): http://localhost:8083


1) Levantar backend + base de datos (Docker)
---------------------------------------------
Desde la raiz del repo:

docker compose up --build -d

Verificar estado:

docker compose ps

Debes ver:
- juegos1000tres-postgres (healthy)
- juegos1000tres-backend-app (running)

Puertos usados por Docker Compose:
- PostgreSQL: 5432
- Backend host: 8083 (mapeado al 8082 interno del contenedor)


2) Levantar frontend Angular
----------------------------
En otra terminal:

cd .\juegos1000tres-frontend
npm install
npm start

Angular queda en:
- http://localhost:4200

Si el 4200 esta ocupado:

ng serve --port 4201


3) Prueba funcional minima
--------------------------
1. Abre el frontend en http://localhost:4200
2. Crea sala o unete a una sala existente.
3. Inicia cada juego desde el lobby:
   - Space Invaders
   - Prueba WebSocket
   - Preguntas
4. Verifica que cada juego carga en Angular (sin iframes).


4) Comandos utiles
------------------
Ver logs del backend:

docker compose logs -f backend

Ver logs de postgres:

docker compose logs -f postgres

Reiniciar solo backend:

docker compose restart backend

Parar todo:

docker compose down

Parar todo y borrar volumen de BD:

docker compose down -v


5) Problemas comunes
--------------------
Si backend no arranca, revisa logs:

docker compose logs -f backend

Si frontend no conecta al backend:
- confirma que backend este arriba en puerto 8083
- confirma que el frontend este en 4200 y sin errores de compilacion

Si hay puertos ocupados, cierra procesos previos y vuelve a lanzar.
